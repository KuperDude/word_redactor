# DOCX Live Editor – живое редактирование шаблонов DOCX в терминале

Инструмент позволяет заполнять DOCX‑шаблоны в консоли с **автообновляемым текстовым превью** (через `tmux` + `doxx`).  
Всё работает внутри Docker – никаких проблем с зависимостями ни на Windows, ни на Linux.


> **Важно:** не меняйте имена папок. Скрипты используют относительные пути `../templates` и `../outputs`.


## 🚀 Запуск

### 1. Сборка образа (один раз)

Откройте терминал в **корневой папке проекта** и выполните:

```bash
docker build -t docx-live-arch .
```

### 2. Запуск

#### Для Windows

Дважды кликните по `run.bat`.

#### Для Linux

```bash
chmod +x run.sh && ./run.sh
```

## 📖 Как пользоваться

1. **Выбор шаблона** – после запуска увидите список файлов в `templates/`. Введите номер.  
   Сразу после выбора в правой панели появится текстовое содержимое шаблона.

2. **Приложения**  
   - Введите количество (0 – если не нужно).  
   - Для каждого: заголовок → страницы (можно пропустить) → текст (пустая строка – конец).  
   Превью обновляется после каждого ввода.

3. **Плейсхолдеры**  
   Скрипт находит все `{Имя_поля}` (кроме `{APPENDICES}`). Введите значения (Enter = пробел).  
   Превью обновляется после каждого ввода.

4. **Сохранение**  
   - Спросит: сохранить финальную версию? (`y`/`n`).  
   - Если `y` – введите имя файла (без расширения). Готовый `.docx` появится в `outputs/`.  
   - Если `n` – последняя версия останется как `outputs/preview.docx`.

5. **Выход** – нажмите `Ctrl+C` в левой панели. tmux‑сессия закроется, вы вернётесь в консоль.

## 📂 Где искать результаты

| Файл | Местоположение |
|------|----------------|
| Промежуточный превью | `outputs/preview.docx` |
| Сохранённые финальные документы | `outputs/ваше_имя.docx` |
| Шаблоны (кладите сюда) | `templates/` (только `.docx`) |

## ⚠️ Возможные проблемы

| Проблема | Решение |
|----------|---------|
| Правая панель пуста после выбора шаблона | Шаблон может не содержать текста (только картинки). `doxx` показывает только текст. |
| tmux жалуется на размер окна | Игнорируйте. Просто увеличьте окно терминала. |
| На Windows не монтируется папка | Убедитесь, что Docker Desktop запущен и имеет доступ к диску (Shared Drives). |
